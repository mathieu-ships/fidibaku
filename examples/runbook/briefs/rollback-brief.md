# Rollback readiness brief

## Objective

Document a rollback path that can be executed quickly and reviewed in the browser with Fidibaku.

## Current state

The example runbook says rollback has been rehearsed. A production runbook should include command ownership, expected duration, and post-rollback validation checks.

## Acceptance criteria

- Rollback command or procedure is linked from the runbook.
- Expected rollback duration is stated.
- Validation checks are listed in order.
- Communication owner is named.
- The brief explains when rollback is preferred over forward-fix.

## Review prompts

- Would a new operator understand the first step?
- Is there a clear decision threshold?
- Are user or stakeholder communications covered?
