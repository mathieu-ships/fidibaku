Fig 0.0 / Execution plan

Build plan & runbook — sequenced work in dispatch batches

Every build-worthy gap is represented as one implementation unit. The table groups work into dependency dispatch batches: Batch 1 can run in parallel; core readiness follows; instrumentation unlocks keyboard and comparison flows; release controls add archive and routing; cadence adds ongoing reminders. Impact estimates how much the work moves the user journey; Value is the raw reviewer/operator value.

Live status: the fictional team has a working review loop and is now hardening assignment, handoff, access, and observability. The goal is a public-safe example that an agent can inspect, comment on, and use as a template.
▸ The goal: make every review comment actionable. The HTML carries stable anchors, the sidecars carry structured feedback, and the linked briefs show the sort of implementation context an agent would read before making changes.
▸ Demo safety: all names, teams, links, and work items are fictional. This file intentionally contains no customer data, private repository names, live credentials, or internal implementation details.
